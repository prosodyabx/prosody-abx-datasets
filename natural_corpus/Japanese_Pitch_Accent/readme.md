# Japanese Natural Corpus (Pitch Accent Contrasts)

This directory contains the natural (human-recorded) Japanese corpus used for pitch-accent contrast analysis in Prosodic ABX experiments.

## Overview

- Speakers: 10 native Japanese speakers who can speak the Kanto dialect
- Design-level core items: 23 kana sequences with potential pitch-accent differences
- Contrast units: 46 pitch-accent contrasts
- Goal: compare pitch-accent realizations under matched sentence context

## Recording Design

For each kana sequence, we designed two sentence pairs.

- Each pair contains two sentences with the same kana sequence but different pitch accent patterns.
- In each pair, one target appears in sentence-initial position and the other appears in sentence-medial position.
- Non-target words are controlled to be the same within a pair to keep context comparable.

Important note:

- The target words in pair A and pair B for the same kana sequence are not always identical.
- During corpus construction, some items were adjusted: some contrasts use different kanji forms, and some include following kana (e.g., for heiban/0-type patterns).
- Therefore, each pair is treated as an independent pitch-accent contrast with matched context, and the released corpus is analyzed based on 46 contrasts.

Total contrasts in this release: 46.

## OSF Upload Note

Audio files (`utterances` and `segments`) are uploaded as `.zip` archives because uploading a very large number of individual files to OSF is slow and may cause upload failures.

Please unzip the archives before use.  
The metadata files (`metadata/*.tsv`) reference utterance/segment IDs by file name.


## Audio Files

- Sample rate: 48kHz

### `audio/utterances/`

Full sentence recordings.

Filename format:

```text
jp_{speaker}_{contrast_index}_{sid}.wav
```

Example:

```text
jp_S001_c01_01.wav
```

- `speaker`: speaker ID (e.g., `S001`)
- `contrast_index`: contrast ID (e.g., `c01`, `c02`, ...)
- `sid`: sentence condition ID within each contrast

### `audio/segments/`

Target segments clipped from full utterances.

Filename format:

```text
jp_{speaker}_{contrast_index}_{sid}_seg.wav
```

Example:

```text
jp_S001_c01_01_seg.wav
```

Word/target boundaries were first generated by MFA and then manually adjusted.

## Metadata

### `metadata/utterances.tsv`

Sentence-level metadata.

- `utterance_id`: full-sentence audio ID (without extension)
- `speaker`: speaker ID
- `contrast`: contrast key
- `word`: target word in the sentence
- `label`: pitch-accent label (accent type)
- `text`: sentence text

Pitch-accent label convention in this corpus:

- `0`: heiban (no downstep within the target word)
- `1`: fall after mora 1 (atamadaka-type pattern)
- `2`: fall after mora 2
- `n`: fall after mora `n` (same rule for larger values)

### `metadata/segments.tsv`

Segment-level metadata.

- `segment_id`: clipped segment ID
- `utterance_id`: parent utterance ID
- `onset`: segment start time (s)
- `offset`: segment end time (s)
- `duration`: segment duration (s)

### `metadata/recording_design.tsv`

Speaker-independent recording design.

- `contrast_index`: contrast ID (`c01`, `c02`, ...)
- `contrast`: contrast key
- `sid`: sentence condition ID
- `word`: target word
- `text`: prompt sentence
- `label`: intended pitch-accent label

## Annotation and Quality Control

- A native Japanese speaker was used for quality control to filter out non-standard pronunciations.
- A very small number of items judged as non-standard pronunciations were excluded.
- Remaining items show standard and clear pitch-accent realizations.
